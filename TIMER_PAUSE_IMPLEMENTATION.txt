1. ESTADOS AGREGADOS AL HOOK useTimer
─────────────────────────────────────────────────────────────────

interface PauseRecord {
  start: string;      // ISO timestamp del inicio de pausa
  end: string | null; // ISO timestamp del fin de pausa
  duration: number;   // duración en segundos
}

Estados nuevos:
• isPaused: boolean - indica si el timer está en pausa
• pausedTime: number - tiempo acumulado de todas las pausas (segundos)
• pauseStart: Date | null - timestamp del inicio de la pausa actual
• pauseHistory: PauseRecord[] - historial completo de pausas

Funciones nuevas:
• handlePauseTimer() - Pausa el timer en curso
• handleResumeTimer() - Reanuda el timer después de pausa

─────────────────────────────────────────────────────────────────
2. LÓGICA DE CÁLCULO IMPLEMENTADA
─────────────────────────────────────────────────────────────────

✓ El interval del timer solo corre cuando: timerRunning && !isPaused
✓ timerSeconds representa el tiempo efectivo de trabajo (sin pausas)
✓ Al pausar: se guarda pauseStart y se detiene el interval
✓ Al reanudar:
  - Se calcula la duración de la pausa
  - Se suma a pausedTime acumulado
  - Se agrega al historial pauseHistory
  - Se reactiva el interval
✓ Al finalizar con pausa activa: se cierra la pausa automáticamente
